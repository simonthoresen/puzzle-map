# Puzzle Design Rules

These rules apply to every puzzle generated by this skill. They are
non-negotiable.

## Core principle

The decoded / solved answer must **BE** the location. Never wrap the answer
in surrounding text that gives it away.

## Rules

1. **No method hints.** Do not write "read this backwards", "shift by 3",
   or "unscramble these letters." The solver must figure out the method.
   The one exception is emoji cues (like `A🔄Z` for Atbash or `⬅️` for
   backwards) — these are acceptable as subtle visual hints.

2. **No zone giveaways.** Do not include floor names, room names, or zone
   labels when the decoded text already implies a specific location.

3. **No fill-in-the-blank giveaways.** Never use a sentence like
   "Look in the ___ drawer" where the blank is trivially guessable from
   surrounding words.

4. **Back-of-note = component, not answer.** The reverse side of a note
   should contain a *piece needed to solve* the front (a cipher key, a
   drawing, a number). Never put the raw answer or a near-answer on the back.

5. **Chained values must be load-bearing.** When a puzzle requires a value
   from a previous step (stair count, computed number), that value must be
   essential to *both* puzzles — not just "remember this for later."

6. **Riddles need misdirection.** A riddle should not make the answer
   completely obvious. Add at least one layer of misdirection or require
   genuine deduction.

7. **Acrostics spell the FULL answer.** Trivia acrostics must spell out the
   entire location word. Never spell a partial word and place it in a
   giveaway sentence.

8. **Answers must be verifiable.** Every puzzle must have exactly one
   correct answer. Ambiguous puzzles frustrate players.

9. **Personal-knowledge puzzles must be solvable by the player.** Only use
   facts the specific player actually knows. Cross-check against the player
   profile.

10. **Cipher texts must decode cleanly.** Double-check every encoded string
    by decoding it yourself before including it. A single wrong letter ruins
    the puzzle.
